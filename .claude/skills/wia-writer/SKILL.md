## 风格规则

### 语言

- 中文写作，专业术语保留英文原词（如 attention、softmax、token），首次出现时用**加粗**标注
- 句子简洁直接，一句话能说清楚的不写两句
- 禁止使用"非常"、"十分"、"极其"等程度副词
- 禁止使用"众所周知"、"不言而喻"等套话
- 每个新概念先说"为什么需要它"，再说"它是什么"

### 数学公式

- 一律使用 LaTeX 格式，行间公式用 `$$...$$`，行内公式用 `$...$`
- 禁止用反引号包裹公式（~~`softmax(QK^T)`~~）
- 公式出现后，必须有文字解释每个符号的含义
- 复杂公式拆步骤讲，不要一次甩出完整推导

**示例（正确写法）：**

$$
\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V
$$

拆开来看，它做了三件事：
| 步骤 | 计算 | 含义 |
|:---|:---|:---|
| ① | $QK^\top / \sqrt{d_k}$ | 计算每对位置之间的相关性并缩放 |
| ② | $\text{softmax}(\cdots)$ | 把相关性转成概率权重 |
| ③ | $\cdots V$ | 按权重把 Value 信息汇总 |

### 代码

- 使用 Python（PyTorch 风格）
- 每一个非显而易见的行都加注释，注释说明"这一行在做什么"而非重复代码
- 变量名使用论文惯例（`d_k`、`n_heads`、`head_dim` 等）
- 如果是简化版伪代码，需在代码块前注明
- 代码段落前后都要有文字衔接，不要孤立出现

**示例（正确写法）：**

```python
def scaled_dot_product_attention(query, key, value, mask=None):
    d_k = query.size(-1)
    # ① 计算相关性分数并缩放，避免数值过大导致 softmax 梯度消失
    scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)
    if mask is not None:
        scores = scores + mask   # -inf 位置在 softmax 后会变为 0
    # ② softmax 得到归一化的注意力权重
    weights = scores.softmax(dim=-1)
    # ③ 用权重对 Value 做加权求和
    return torch.matmul(weights, value), weights
```
